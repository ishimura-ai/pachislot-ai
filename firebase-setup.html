Firebase セットアップガイド

全国のユーザーデータをリアルタイム収集するためのFirebase設定（所要時間：約10分）

⚠️
友人のデータが見えない理由 現在はLocalStorage（ブラウザ内）にデータを保存しているため、あなたのPC以外で行われた解析はダッシュボードに届きません。 Firebaseを設定すると、全国どこからでもデータが自動でダッシュボードに集まります。
✅
Firebase無料枠で十分 Firebase Firestoreの無料枠（Sparkプラン）は、読み取り50,000回/日・書き込み20,000回/日で永久無料です。 友人数十人が毎日使っても余裕で収まります。
1
Googleアカウントでfibase.google.comにアクセス
⏱ 1分

Googleアカウントにログインした状態で、下のボタンをクリックしてFirebaseコンソールを開きます。

🔥 Firebaseコンソールを開く（新しいタブ）
2
新しいプロジェクトを作成
⏱ 3分

Firebaseコンソールで＋ プロジェクトを追加をクリックします。

📸 [スクリーンショット：「プロジェクトを追加」ボタン]

プロジェクト名に pachislot-ai-data と入力して次へ
Googleアナリティクスの設定画面はスキップしてOK（無効のまま「プロジェクトを作成」）

3
Firestoreデータベースを有効化
⏱ 2分

左サイドバーから🔥 Firestore Databaseをクリック → データベースを作成ボタンをクリック。

モードの選択：「テストモードで開始」を選択（30日間は制限なし、後で変更可）
ロケーション：asia-northeast1（東京） を選択 → 「有効にする」

📸 [スクリーンショット：Firestoreのモード選択画面]
4
セキュリティルールを設定（書き込み許可）
⏱ 2分

Firestoreのルールタブをクリック。
以下のルールをコピーして貼り付け、公開ボタンをクリックします。

// Firestoreルール - 誰でも書き込み可能（管理者のみ読み取り） rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /hall_data/{document} { allow create: if true; // 全員書き込みOK allow read, update, delete: if false; // 読み取りは禁止（管理者のみFirebase Adminで読む） } } }

⚠️ 上記ルールはユーザーは「書き込みだけ」できます。読み取りは管理者（あなた）がAPIキーで行うため、他のユーザーは他人のデータを見られません。

5
ウェブアプリを登録してAPIキーを取得
⏱ 2分

プロジェクトの設定（歯車アイコン → プロジェクトの設定）を開き、
「マイアプリ」セクションから</>（ウェブ）アイコンをクリック。
アプリのニックネームに「pachislot-web」と入力してアプリを登録。

次の画面に firebaseConfig が表示されます。
この値を下のフォームに貼り付けてください：

const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "pachislot-ai-data.firebaseapp.com", projectId: "pachislot-ai-data", storageBucket: "pachislot-ai-data.appspot.com", messagingSenderId: "...", appId: "..." };

🎉 設定が完了しました！

PACHISLOT AIが自動的にFirebaseに接続されました。
これからアプリで解析を行うと、全台のデータがFirebaseに自動送信されます。

友人に改めてアプリを使ってもらうと、30秒以内にダッシュボードに表示されます。

📊 ダッシュボードを開く 🎰 アプリに戻る
6
アプリを友人と共有する
⏱ 1分

Firebaseを設定したら、あとはアプリ（index.html）を友人にシェアするだけです。
友人がどこにいても、解析するたびに自動で全データがダッシュボードに届きます。
